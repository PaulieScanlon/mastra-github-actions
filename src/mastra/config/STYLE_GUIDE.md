# Blog Style Guide

This guide focuses on objective issues only. Do not flag stylistic preferences like voice, sentence length, or tone.

## Required Checks

### Frontmatter
- Must include: title, publishedAt, summary, author, draft, categories
- publishedAt must be valid date format (YYYY-MM-DD)

### Headings
- Use title case for H1/title (capitalize first letter of each word)
- Use sentence case for H2 and below (capitalize first word only)
- Dates in titles (e.g., "2026-01-20") are exempt from case rules
- No skipping heading levels (e.g., H2 followed by H4)

### Images
- All images must have alt text
- Image paths must be valid

### Links
- No broken links
- No empty link text

### Code Blocks
- Must have a language specified

### Spelling
- Check for obvious spelling errors
- Ignore technical terms, product names, and code

## Do NOT Flag

- Passive vs active voice
- Sentence length or complexity
- Use of contractions
- Informal tone or emoji usage
- Sentence fragments (often intentional)
- Oxford comma usage
- Paragraph length
